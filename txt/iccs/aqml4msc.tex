\section{Introduction}



	% Multisource
	Data about an observed object can be gathered from multiple complementary sensors.
	To obtain richer and more accurate decision outcomes, multisource information fusion (MSIF) can be employed.
	A comprehensive review of information fusion approaches is provided by Li et
	al.~\cite{liMultisourceInformationFusion2024}.\todo{TR: To chyba lepiej na koniec.} In this work, we focus on
	feature-level fusion, in which the fusion model operates on extracted feature representations.
	MSIF has found widespread application in machine learning, particularly in classification tasks, as surveyed
	in~\cite{baltrusaitisMultimodalMachineLearning2019}.
	Representative applications include remote sensing~\cite{liA3CLNNSpatialSpectral2022}, military
	systems~\cite{sycaraIntegratedApproachHighlevel2009},
	and human action recognition~\cite{fortinoAdvancesMultisensorFusion2019}, among
	others~\cite{liMultisourceInformationFusion2024}.\todo{TR: Wstęp bardzo do rzeczy, ale ok.}
	
	
	% Transition: MSIF -> QML
	Despite the success of classical machine learning models for feature-level fusion, their performance can be limited
	by model capacity and scaling issues in high-dimensional feature spaces.\todo{TR: Mamy źródło?}  This motivates the
	exploration of alternative computational paradigms for multisource learning.
	
	
	% QML
	Quantum machine learning (QML) leverages principles of quantum mechanics to process information in ways that differ
	fundamentally from classical machine learning~\cite{biamonteQuantumMachineLearning2017a}. \todo{TR: Tutaj zdanie o
	tym jak QML wygląda: że używa obwodów kwantowych do wyrażania obliczeń.}
	Several works have demonstrated that quantum models can represent complex functions using comparatively shallow
	circuits or fewer parameters than their classical counterparts, suggesting potential advantages for learning tasks
	involving structured or high-dimensiona data~\cite{havlicekSupervisedLearningQuantumenhanced2019,
	schuldQuantumMachineLearning2019,abbasPowerQuantumNeural2021}. Moreover, theoretical work has identified learning
	scenarios in which quantum models offer data-dependent advantages~\cite{huangPowerDataQuantum2021a}.
	
	
	% VQA on NISQ 
	As fault-tolerant quantum computers have not yet been realized \todo{TR: Ale są całkiem mocne ambicje i prace z
	ostatniego czasu. Można wspomnieć.}, practical QML research focuses on algorithms compatible with current noisy
	intermediate-scale quantum (NISQ) devices~\cite{chenComplexityNISQ2023}. Among these, variational quantum
	algorithms (VQAs)~\cite{cerezoVariationalQuantumAlgorithms2021} are particularly prominent. VQAs are hybrid
	quantum--classical methods that employ a classical optimizer to train a parameterized quantum circuit. Importantly,
	Schuld et al.~\cite{schuldCircuitcentricQuantumClassifiers2020a} have shown that such models exhibit robustness to
	noise in both input data and circuit parameters, making them suitable for near-term quantum hardware.
	
	
	% QMSC
	Recently, several works have begun to explore the application of quantum machine learning methods to multisource or
	multimodal data fusion. Feature-level multimodal quantum fusion presented in pre-print from
	2026~\cite{wuFeatureEntanglementbasedQuantum2026} takes features extracted by sperate unimodal classical neural
	networks. Extracted features are embedded into quantum fusion circuit and entangled between multiple modalities and
	target qubits, with trainable parameters. Their model showed consistent stability and high accuracy for
	high-dimensional data.

\section{Problem Formulation} \label{sec:problem_formulation}


	We consider a synthetic multisource classification setting, where multiple input sources are derived from a single
	original sample.\todo{TR: To jeszcze się okaże.}  Specifically, an input image is partitioned into two
	complementary views. Each view is independently transformed by a deterministic preprocessing pipeline into a vector
	representation, denoted as $x_{\mathrm{top}}$ and $x_{\mathrm{bottom}}$, which constitute the synthetic multisource
	inputs used for classification.
	
	The vector inputs are subsequently passed through learnable feature extraction functions $f(\cdot,\theta_1)$ and
	$f(\cdot,\theta_2)$, producing feature vectors $v_{\mathrm{top}}$ and $v_{\mathrm{bottom}}$ of equal dimensionality.
	These features are fused and classified by a function $h(\cdot,\theta_3)$, yielding the final classification
	decision $d$. The resulting formulation is defined as:
	\begin{align*}
		v_{\mathrm{top}}    & = f(x_{\mathrm{top}}, \theta_1),                      \\
		v_{\mathrm{bottom}} & = f(x_{\mathrm{bottom}}, \theta_2),                   \\
		d                   & = h(v_{\mathrm{top}}, v_{\mathrm{bottom}}, \theta_3).
	\end{align*}
	All learnable parameters $\theta_1$, $\theta_2$, and $\theta_3$ are optimized jointly
	during training.

	The MNIST dataset consists of 70,000 images of handwritten digits (0--9), each of size $28 \times 28$ pixels.
	The dataset was obtained from Kaggle \cite{lecunMNISTHandwrittenDigits2025}, and we used the predefined
	training--validation split provided with the dataset without modification.
	
	From the dataset, we select only three digit classes:\todo{TR: Dlaczeo akurat te? Arbitralnie też jest powodem.
	Różnorodność jest ok. Ale coś powiedzieć trzeba.}
	\begin{itemize}
		\item digit ``5'' (\(5421\) training samples and \(892\) validation samples),
		\item digit ``6'' (\(5918\) training samples and \(958\) validation samples),
		\item digit ``7'' (\(6265\) training samples and \(1028\) validation samples).
	\end{itemize}
	Each image is preprocessed to construct synthetic multisource inputs. Specifically, images are split horizontally
	into top and bottom halves, which are treated as complementary views. Each view is then binarized, spatially
	downsampled, and transformed into a six-dimensional vector by computing the mean pixel intensity along columns.
	These vectors correspond to the synthetic multisource inputs $x_{\mathrm{top}}$ and $x_{\mathrm{bottom}}$ defined
	in Section~\ref{sec:problem_formulation}.\todo{TR: Jeśli umiesz dodać uzasadnienie, to byłoby wskazane.}

			The angle embedding is defined in Eq.~\ref{eq:embedding}, where \(n\) denotes the number of qubits—equal to
			the number of features fed into the classifier, and \(v_i\) is the \(i\)-th component of the feature vector

				\ket{\phi(v)} & = 	
					U_{\mathrm{emb}}(v)\ket{0}    \\
     	         & = \left( R_X(v_1) \otimes R_X(v_2) \otimes \cdots \otimes R_X(v_n) \right) \ket{0}. \nonumber

			Here,
			\begin{equation*}
				R_X(\alpha) = e^{-\mathrm{i} \alpha \sigma_X / 2}.
			\end{equation*}\todo{Jeszcze nie zdefiniowałeś (chyba), że $\sigma_\square$ to macierze Pauliego. W tym

			\tr{we used} is defined as
			\begin{equation}
				\boldsymbol{\Theta}
				=
				\left\{
				\boldsymbol{\theta}^{(1)}, \boldsymbol{\theta}^{(2)}, \ldots, \boldsymbol{\theta}^{(L)}
				\right\},
				\qquad
				\boldsymbol{\theta}^{(\ell)} \in \mathbb{R}^{n},
			\end{equation}
			where \(L\) denotes the number of entangler layers and \(n\) is the number of qubits. The overall ansatz
			unitary is given by
			\begin{equation}
				U_{\mathrm{ansatz}}(\boldsymbol{\Theta})
				=
				\prod_{\ell=1}^{L}
				U_{\mathrm{BEL}}(\boldsymbol{\theta}^{(\ell)}),
			\end{equation}
			\tr{where $U_{BEL}$ is a unitary matrix representing} a single Basic Entangler Layer\tr{. It's} defined as
			\begin{equation}
				U_{\mathrm{BEL}}(\boldsymbol{\theta}^{(\ell)})
				=
				\left(
				\prod_{i=1}^{n}
				\mathrm{CNOT}_{i,\,i+1 \bmod n}
				\right)
				\left(
				\bigotimes_{i=1}^{n}
				R_Y(\theta^{(\ell)}_i)
				\right),
			\end{equation}
			where
			\begin{align*}
				R_Y(\theta) & = e^{-\mathrm{i} \theta \sigma_Y / 2}, \\
				\mathrm{CNOT}
				            & =
				\ket{0}\!\bra{0} \otimes \mathbb{I}
				+
				\ket{1}\!\bra{1} \otimes \sigma_X.
			\end{align*}
			Applying the ansatz unitary to the embedded quantum state yields
			\begin{equation}
				\ket{\gamma}
				=
				U_{\mathrm{ansatz}}(\boldsymbol{\Theta}) \ket{\phi}.
			\end{equation}
			At the output of the circuit, a vector of expectation values \tr{of the Pauli-\(Z\) operator on the first
			three qubits are obtained by the final state measurement}. Each expectation value corresponds to the

			expectation value for qubit \(j\) is given by
			\begin{equation}
				\langle \sigma_Z \rangle_{\gamma, j}
				=
				\bra{\gamma} \sigma_Z^{[j]} \ket{\gamma},
				\qquad j = 1,2,3,
			\end{equation}
			where
			\begin{equation}
				\sigma_Z^{[j]}
				=
				\mathbb{I}^{\otimes (j-1)} \otimes \sigma_Z \otimes \mathbb{I}^{\otimes (n-j)}.
			\end{equation}
